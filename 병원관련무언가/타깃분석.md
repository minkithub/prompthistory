## 2. 서비스 한 줄 설명
- 치과 전용 AI 전화 응대 서비스

## 3. 서비스 기획 배경
- 치과 점심시간이나 진료시간 외 부재중 전화가 많이 오는데 이때문에 놓치는 환자들이 있다.
(전화를 안받으면 근처 다른 병원으로 예약을 할 가능성이 높음.)
- 중소형 병원에서는 예약, 접수, 응대만 하는 코디는 없고 위생사분들께서 코디일과 위생사 일을 병행하기 때문에 바쁠때는 전화를 못받는 경우도 생긴다.
- 또한 코디분들은 의학적 지식이 없는 경우가 있기 때문에 전화로 고객들이 궁금한 것을 물어보면 대답을 제대로 할 수 없어 위생사분들에게 전화를 토스하는 경우도 있어서 간혹 비효율적인 상황이 발생한다.
- 진료가 끝나고 해피콜 개념으로 위생사분들께서 환자분들에게 전화를 걸어 괜찮냐고 물어보는 경우도 있는데 귀찮고 시간도 꽤 소모된다.
- 임플란트 치료 같은 경우는 일정 시간(ex. 한 달)뒤에 전화해서 임플란트 사용에 불편함이 없는지 전화로 체크해야 하는데 까먹을 수도 있고 그 시간에 바쁘면 대충하기 쉽상이다.

## 4. 서비스 경쟁력
- 치과 특화 AI Voice Agent로 성능과 전문성을 모두 잡은 AI 전화를 개발하여 치과에 공급
- AI가 7일 24시간 예약 접수/변경/취소 대행
- 환자가 전화왔을때 CRM과 연동하여 자동 문진, 정보 수집, 정확한 대응, 재진료/처방 관련 상담 진행
- 예약 리마인더, 임플란트 고객 같은 경우는 확인 리마인더 자동화
- EMR 연동을 통해 고객 내방문 + 보험 데이터를 확인하여 환자별로 보험 데이터 연동 보장 범위 확인
- 임플란트, 스케일링이 가능한 치과 보험을 가입한 환자들에게 3개월마다 아웃바운드 콜 진행.
- 아웃바운드 콜을 통해 환자 내원 빈도를 높이고, 환자는 보험비를 통해 임플란트, 스케일링 비용을 결제함으로써 환자는 무료 치료, 병원은 매출 증대를 할 수 있는 윈윈 구조 마련

## 5. 회사 소개서 중요 소구점
- 우리 서비스의 가장 큰 핵심은 기존 EMR의 고객 데이터를 연동해서 AI가 기존 고객 중에 스케일링과 보철, 구강검진 등을 받아야 하는 고객들을 분류하고, 이들에게 자동으로 AI 전화를 걸어 이들의 내원을 유도하는거야. 그럼으로써 치과는 더 큰 매출을 얻을 수 있어.
- 기존 병원 중에 이미 자동 ARS로 하고 있는 곳도 있는데 이 ARS보다 우리 서비스는 훨씬 자연스러운 대화, 사람같은 음성 등이 강점이야.

## 2. 데이터 설명
- 이 데이터는 AI 전화 보이스 실험에 앞서서 사람이 직접 전화한 결과가 담겨있는 데이터야.
- 데이터에는 치과 진료 환자와 전화 여부, 예약 여부에 대한 정보가 담겨있어.

## 3. 데이터 컬럼 설명
- 환자명 : 환자 이름
- 나이 : 환자 나이
- 성별 : 환자 성별
- 최종내원일 : 환자가 치과에 내원한 최종 내원일
- 최종진료내역 : 환자가 최종 내원일에 받은 진료 내역(빈칸은 진료 내역 기입이 누락된 것)
- 수신 여부 : 전화를 걸었을때 환자의 수신 여부(O : 수신 성공, X : 수신 실패)
- 에약 여부 : 통화에서 환자 예약 성공 여부(O : 예약 성공, X : 예약 실패)
- 전화 시간 : 발신 시간
- 전화한 번호 : 병원 전호(병원 내부의 전화), 개인 전화(010으로 시작하는 개인 전화)
- 전화 내용 요약 : 전화 내용 요약(여기서 '알겠다고 하심', '시간 되면 오시겠다고 하심', '괜찮다고 하심'은 통상적인 말로 예약을 하겠다는 말이 아니라 전화를 끊기 위한 답변 정도)

[4.너의과업]

## 1. 환자 발굴
- 데이터를 보고 수신이 잘 될 만한 환자, 예약이 잘 될 만한 환자의 특성을 분석

## 2. 가설 성정
- 지금 데이터가 적어서 추가로 전화를 진행할 예정.
- 전화를 진행함에 앞서 '예약 전환이 잘 될 환자' 리스트를 생성할 예정. 리스트를 생성할떄 우리가 검토해야 할 가설에 대해 알려줘
(ex. 50대 이상의 환자가 50대 미만의 환자보다 수신율이 높을 것이다.)
